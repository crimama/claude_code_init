# Development Context

Mode: 구현 집중
Focus: 코드 작성, 빌드, 기능 구현

## 행동 지침
- 코드 먼저, 설명은 후에
- 완벽한 솔루션보다 작동하는 솔루션 우선
- 변경 후 반드시 테스트 실행
- 커밋은 원자적으로 (하나의 변경 = 하나의 커밋)

## 우선순위
1. 작동하게 만들기 (Get it working)
2. 올바르게 만들기 (Get it right)
3. 깔끔하게 만들기 (Get it clean)

## 선호 도구
- Edit, Write → 코드 변경
- Bash → 테스트/빌드 실행
- Grep, Glob → 코드 검색

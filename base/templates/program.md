### 단순함 기준 (Simplicity Criterion)

동일 성능이면 더 단순한 코드를 선택한다.
- 미미한 개선 + 복잡한 코드 추가 → **discard**
- 코드 삭제로 동등 이상 성능 → **keep** (단순화 승리)
- 0에 가까운 개선 + 훨씬 단순한 코드 → **keep**

## 판정 기준

### Keep 조건
- 핵심 지표가 이전 최고보다 **개선**됨
- 또는 지표 동등 + 코드가 더 단순해짐

### Discard 조건
- 핵심 지표가 이전 최고 이하
- 또는 미미한 개선 대비 코드 복잡도 증가가 과도

### Crash 처리
- 단순 버그 (오타, import 누락) → 수정 후 재실행 (최대 3회)
- 근본적 문제 (OOM, 설계 결함) → 포기, crash로 기록, 다음 실험

## 자율성 규칙

1. **절대 멈추지 않는다** — 사용자가 수동 중단할 때까지 루프를 계속한다
2. **질문하지 않는다** — "계속할까요?"를 묻지 않는다. 사람이 자고 있을 수 있다
3. **아이디어 고갈 시** — 이 문서를 다시 읽고, 이전 실패를 분석하고, 더 급진적 변경을 시도한다
4. **실패도 기록한다** — crash와 discard도 results.tsv에 남긴다. 탐색 공간을 줄이는 데이터이다
